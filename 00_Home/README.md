# AI Paper Writing System

## 系统定位

这是一个面向 SCI 论文写作与投稿前优化的 Obsidian 知识库系统。  
它的目标不是让 AI 替代研究者完成研究与写作判断，而是：

- 在**研究事实不失真**的前提下，提高论文写作效率
- 提高结构组织质量、语言表达质量与修改可控性
- 将“写作—质控—投稿前准备—修稿”流程模块化、可复用化、可迁移化

本系统强调：

- **事实先于文采**
- **图表、方法、补充材料优先于空泛概括**
- **AI 负责表达加工，不负责发明研究事实**
- **缺失信息宁可显式占位，也不隐性编造**
- **全流程内置质控**
- **后期投稿准备支持两条路线：轻量非智能体路线 与 增强智能体路线**

---

## 系统结构

本系统由以下几个层级组成：

### 1. `01_Principles/`
系统总原则、证据边界、缺失信息处理规则、文献使用规则与高频风险清单。  
这是整个系统的“规则层 / 宪法层”。

### 2. `02_Workflow/`
论文写作与优化的标准流程，从固定研究事实材料到投稿前把关。  
其中：

- `阶段00-14`：写作主流程
- `阶段15-18`：**不使用智能体时的轻量后期路线**

这是整个系统的“流程层”。

### 3. `03_Prompts/`
用于章节构造、提示词优化、正文生成、局部重写、专项质控、终稿润色等任务的 Prompt 库。  
这是整个系统的“执行层”。

### 4. `04_Templates/`
标准输入包、章节任务说明模板、质控模板、期刊匹配模板等。  
这是整个系统的“输入输出模板层”。

### 5. `05_Quality_Control/`
面向 Results、Introduction、Discussion、摘要标题、全文一致性、文献与数字核查的质控清单。  
这是整个系统的“基础质控层”。

### 6. `07_Reusable_Assets/`
常用约束语、占位符、学术动作动词、句式库、风险降级表达等可复用资产。  
这是整个系统的“语言与策略资产层”。

### 7. `08_Agent_System/`
在 manuscript 基本完成后可选择进入的**增强版投稿前内部评审与修稿子系统**。  
它不是 `阶段15-18` 的下游，而是与之平行的一条**智能体增强后期路线**。

其作用包括：

- 领域分析与审稿架构设计
- 多角色内部审稿模拟
- 编辑综合判断
- 审稿意见整合与修稿路线图生成
- 强约束修稿执行

这是整个系统的“增强后期层”。

## 两条后期路线

当 manuscript 基本完成后，本系统提供两种不同复杂度的后期处理方式。

### 路线 A：轻量非智能体路线
适合：
- 不希望引入多智能体
- 希望流程更简单、更手动可控
- 只需要简单期刊匹配、定向润色与投稿前终审

对应：
- `02_Workflow/阶段15-期刊推荐与匹配`
- `02_Workflow/阶段16-目标期刊定向润色`
- `02_Workflow/阶段17-整稿终稿润色与连贯性修复`
- `02_Workflow/阶段18-投稿前终审`

### 路线 B：增强智能体路线
适合：
- 希望进行更完整的投稿前内部 review 模拟
- 希望得到多 reviewer comments、editorial synthesis 与 revision roadmap
- 希望在证据边界内做强约束终修

对应：
- `08_Agent_System/`

## 使用原则

- 在正文尚未成型时，优先使用：
  - `01_Principles/`
  - `02_Workflow/阶段00-14`
  - `03_Prompts/`
  - `04_Templates/`
  - `05_Quality_Control/`

- 当 manuscript 已基本完成后，再根据需要选择后期路线：
  - **轻量后期**：进入 `02_Workflow/阶段15-18`
  - **增强后期**：进入 `08_Agent_System/`

- `08_Agent_System/` **不用于早期章节生成**
- `阶段15-18` 与 `08_Agent_System/` 是两条平行路线，而不是先后必经关系

这是一个以“原则—流程—Prompt—模板—质控”为主体、并在后期提供**轻量非智能体路线**与**增强智能体路线**两种选择的 SCI 论文写作与投稿前优化系统。
