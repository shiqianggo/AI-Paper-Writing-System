
# 系统地图

本系统由“写作主系统”与“后期投稿准备系统”两大部分组成。  
其中，后期投稿准备系统又分为两条路线：

- **轻量非智能体路线**
- **增强智能体路线**

---

## 一、写作主系统

写作主系统负责从研究事实整理，到章节生成、质控、全文核查的全过程。

```text
01_Principles
   ↓
02_Workflow（阶段00-14）
   ↓
04_Templates
   ↓
03_Prompts
   ↓
05_Quality_Control
   ↓
07_Reusable_Assets（贯穿支持）
```

---

## 二、写作主系统各模块功能

### `01_Principles/`

提供上位规则：

- 事实边界
- 证据边界
- 缺失信息处理
- 文献使用规则
- 高频风险防控

### `02_Workflow/阶段00-14`

提供写作主流程：

- 固定研究事实材料
- 人工写章节任务说明
- 优化章节 Prompt
- 写 Results / Introduction / Discussion / Abstract
- 做章节质控
- 做全文一致性、文献、数字与事实检查

### `04_Templates/`

提供标准输入输出模板，帮助把任务结构化。

### `03_Prompts/`

提供可直接执行的任务型 Prompt：

- 前置构造类
- 写作类
- 质控类
- 去 AI 味与研究特异性增强类
- 文献、数字、终审类

### `05_Quality_Control/`

提供清单式基础质控，用于对章节与整稿做系统检查。

### `07_Reusable_Assets/`

提供高频约束语、句式库、风险降级表达、占位符等复用资产。

---

## 三、后期投稿准备系统

当 manuscript 已基本完成，并完成基础质控后，可以进入后期投稿准备阶段。

这时系统提供两条路线：

---

## 路线 A：轻量非智能体后期路线

适合：

- 不使用智能体的用户
- 希望手动控制流程的用户
- 只需要简单期刊匹配、简单定向润色与基础投稿前终审的用户

对应路径：

![*.txt](https://registry.npmmirror.com/@lobehub/assets-fileicon/1.0.0/files/assets/document.svg)

Plaintext

```
02_Workflow/阶段15-期刊推荐与匹配   ↓02_Workflow/阶段16-目标期刊定向润色   ↓02_Workflow/阶段17-整稿终稿润色与连贯性修复   ↓02_Workflow/阶段18-投稿前终审
```

### 这条路线解决什么

- 基础期刊推荐与匹配
- 目标期刊定向调整
- 整稿语言与连贯性修复
- 投稿前最终检查

---

## 路线 B：增强智能体后期路线

适合：

- 希望在投稿前进行更完整内部 review 模拟的用户
- 希望得到多 reviewer comments、editorial-style judgment、revision roadmap 的用户
- 希望在证据边界内做更系统终修的用户

对应路径：

```
08_Agent_System   ├── Field Analysis & Review Architecture Agent   ├── Multi-Perspective Review Board Agent   ├── Editorial Synthesis & Decision Agent   ├── Revision Planning & Comment Consolidation Agent   └── Constrained Manuscript Revision Agent
```

### 这条路线解决什么

- 领域分析与审稿架构设计
- 多角色审稿模拟
- 编辑综合判断
- 评论整合与修稿规划
- 强约束修稿执行

---

## 四、两条后期路线的关系

两条路线的关系不是：


```
阶段15-18 → 08_Agent_System
```

而是：


```
阶段00-14 完成后   ↓   ├── 进入 阶段15-18（轻量非智能体路线）   └── 进入 08_Agent_System（增强智能体路线）
```

也就是说：

- `阶段15-18` 不是 Agent 系统的前置依赖
- `08_Agent_System` 不是 `阶段15-18` 的附属扩展
- 二者是**平行的后期分支**

---

## 五、Prompt、QC、Agent 三者关系

### Prompt

适合：

- 单次任务
- 局部生成
- 局部改写
- 局部检查

### Quality Control

适合：

- 清单式核查
- 基础风险过滤
- 章节与整稿层面的系统排错

### Agent System

适合：

- 多角色串联审稿
- 编辑综合决策
- 审稿意见整合
- 修稿路线规划
- 强约束修稿执行

---

## 六、最简理解

如果把整个系统看成一篇论文从材料走向投稿的过程，那么：

- `01_Principles/` 是规则
- `02_Workflow/阶段00-14` 是写作主线
- `03_Prompts/` 是任务执行工具
- `04_Templates/` 是结构化接口
- `05_Quality_Control/` 是基础检查层
- `07_Reusable_Assets/` 是表达支持库
- `02_Workflow/阶段15-18` 是轻量非智能体后期路线
- `08_Agent_System/` 是增强智能体后期路线

---

## 七、一句话总结

本系统的核心结构是：
```
先按原则与流程完成写作主系统→ 再在后期阶段从“轻量非智能体路线”和“增强智能体路线”中选择一条
```
