# 使用路径建议

本系统支持两类后期使用方式：

- **常规路径**：不使用多智能体，依靠 Workflow + Prompts + QC 完成写作、润色、期刊匹配与投稿前把关
- **增强路径**：在手稿基本完成后，进入 `08_Agent_System/`，进行多角色内部评审、编辑综合决策与强约束修稿

你可以把它理解为：

- `02_Workflow/阶段00-14` 是写作主流程
- `02_Workflow/阶段15-18` 是**非智能体用户的轻量后期路径**
- `08_Agent_System/` 是**使用智能体用户的增强后期路径**

---

## 路径 A：第一次完整搭建项目

适合：

- 第一次使用本系统
- 想完整走一遍标准流程
- 希望先理解“原则—流程—模板—Prompt—质控”的基本结构

推荐路径：

1. [[01_Principles/总原则]]
2. [[01_Principles/证据边界规则]]
3. [[02_Workflow/阶段00-固定研究事实材料]]
4. [[04_Templates/Template-标准输入包]]
5. [[03_Prompts/Prompt-章节任务说明重构]]
6. [[03_Prompts/Prompt-写作提示词优化]]
7. 对应章节写作 Prompt
8. 对应章节质控清单
9. 全文一致性、文献、数字与终审清单

---

## 路径 B：我已经有材料，只想快速写 Results

适合：

- 图表、方法、补充材料已齐
- 想优先生成 Results
- 暂时不处理全文与投稿前问题

推荐路径：

1. [[04_Templates/Template-Results输入模板]]
2. [[03_Prompts/Prompt-章节任务说明重构]]
3. [[03_Prompts/Prompt-写作提示词优化]]
4. [[03_Prompts/Prompt-Results写作]]
5. [[05_Quality_Control/Results质控清单]]

---

## 路径 C：我已经有初稿，想做 AI 辅助修订

适合：

- 已有章节草稿
- 主要目标是修正问题、降级风险、提升质量
- 还没有进入完整投稿前模拟审稿阶段

推荐路径：

1. [[05_Quality_Control/Results质控清单]] / 对应章节清单
2. [[03_Prompts/Prompt-去AI味重写]]
3. [[03_Prompts/Prompt-增强研究特异性]]
4. [[03_Prompts/Prompt-全文一致性检查]]
5. [[03_Prompts/Prompt-文献检查]]
6. [[03_Prompts/Prompt-数字统计检查]]
7. [[05_Quality_Control/投稿前终审清单]]

---

## 路径 D：我已经接近终稿，只想做投稿前把关（不使用智能体）

适合：

- 章节基本完成
- 需要做最后质控和期刊适配
- 不想进入多智能体系统
- 只想做简单的期刊匹配、定向润色和投稿前终审

推荐路径：

1. [[05_Quality_Control/全文一致性质控清单]]
2. [[05_Quality_Control/文献与引用质控清单]]
3. [[05_Quality_Control/数字统计与事实质控清单]]
4. [[02_Workflow/阶段15-期刊推荐与匹配]]
5. [[02_Workflow/阶段16-目标期刊定向润色]]
6. [[02_Workflow/阶段17-整稿终稿润色与连贯性修复]]
7. [[02_Workflow/阶段18-投稿前终审]]

### 说明

这一路径适合不启用智能体的用户。  
其中：

- `阶段15` 负责基础期刊匹配
- `阶段16` 负责目标期刊定向润色
- `阶段17` 负责整稿层面的语言与连贯性修复
- `阶段18` 负责投稿前最终检查

这是一条**轻量、可手动控制、低复杂度**的后期路径。

---

## 路径 E：我已经接近终稿，想使用多智能体做增强版投稿前评审与修稿

适合：

- manuscript 已基本完整
- 已完成基础质控
- 希望在正式投稿前模拟编辑部与 reviewer 系统
- 希望获得更系统的审稿意见整合与修稿路线图
- 希望在强约束下推进最后一轮修订

推荐路径：

1. [[05_Quality_Control/全文一致性质控清单]]
2. [[05_Quality_Control/文献与引用质控清单]]
3. [[05_Quality_Control/数字统计与事实质控清单]]
4. [[08_Agent_System/README]]
5. [[08_Agent_System/Agent总览与调用顺序]]
6. [[08_Agent_System/Agent工作流映射]]
7. [[08_Agent_System/Agent输入输出接口规范]]

然后按固定顺序进入 5 个 Agent：

1. Field Analysis & Review Architecture Agent
2. Multi-Perspective Review Board Agent
3. Editorial Synthesis & Decision Agent
4. Revision Planning & Comment Consolidation Agent
5. Constrained Manuscript Revision Agent

### 说明

这一路径不是替代写作流程，而是用于**成稿后的增强版后期处理**。  
它比 `阶段15-18` 更重、更系统、更接近真实投稿前内部 review 流程。

你可以把两者理解为：

- `阶段15-18` = 非智能体版、轻量版后期流程
- `08_Agent_System/` = 智能体版、增强版后期流程

## 什么时候不要过早进入 `08_Agent_System/`

以下情况不建议进入：

- 还在写 Results 初稿
- Introduction 结构还没定
- Discussion 还只是通用框架
- 文献、数字、统计还没有做基础核查
- 图表、Methods、Supplementary 仍不稳定
- 全文尚未形成较稳定版本

此时应优先留在：

- `02_Workflow/阶段00-14`
- `03_Prompts/`
- `04_Templates/`
- `05_Quality_Control/`
