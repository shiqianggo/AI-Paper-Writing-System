# Workflow README

## 目录定位

`02_Workflow/` 是本系统的流程层。  
它用于规定：从研究事实材料整理，到章节生成、专项质控、全文检查、投稿前处理的标准顺序。

这个目录的核心作用不是提供某一个具体 Prompt，而是回答：

- 现在应该先做什么？
- 下一步应该做什么？
- 哪些步骤属于正文写作？
- 哪些步骤属于质控？
- 哪些步骤属于后期投稿准备？

---

## 本目录包含什么

本目录分为两大部分：

### 第一部分：写作主流程（阶段00-14）
适用于：
- 从原始材料出发搭建 manuscript
- 按章节推进写作
- 对生成内容做专项质控与全文检查

包括：

1. `阶段00-固定研究事实材料`
2. `阶段01-人工写章节任务说明`
3. `阶段02-优化章节提示词`
4. `阶段03-写Results`
5. `阶段04-Results专项质控`
6. `阶段05-写Introduction`
7. `阶段06-Introduction专项质控`
8. `阶段07-写Discussion`
9. `阶段08-Discussion专项质控`
10. `阶段09-写Abstract-Title-Keywords`
11. `阶段10-摘要与标题专项质控`
12. `阶段11-去AI味与增强研究特异性`
13. `阶段12-全文一致性检查`
14. `阶段13-参考文献与引用检查`
15. `阶段14-数字统计与事实检查`

### 第二部分：轻量后期路线（阶段15-18）
适用于：
- manuscript 已接近终稿
- 不使用多智能体系统
- 只想完成基础期刊匹配、目标期刊定向润色、整稿修复与投稿前终审

包括：

16. `阶段15-期刊推荐与匹配`
17. `阶段16-目标期刊定向润色`
18. `阶段17-整稿终稿润色与连贯性修复`
19. `阶段18-投稿前终审`

## 推荐理解方式

最推荐的理解方式是：

### 阶段00-14
这是写作主流程。  
负责把论文从“材料”推进到“基本成稿 + 基础质控完成”。

### 阶段15-18
这是**非智能体用户的轻量后期路线**。  
适合不进入 `08_Agent_System/` 的用户。

---

## 写作主流程（阶段00-14）怎么理解

### 阶段00：固定研究事实材料
先锁定：
- Figures and legends
- Methods
- Supplementary materials
- validated results
- key numbers

### 阶段01：人工写章节任务说明
先由人写清楚这一章节要完成什么，不直接让 AI 自由发挥。

### 阶段02：优化章节提示词
对原始任务说明进行结构化和约束化处理。

### 阶段03 / 05 / 07 / 09：章节写作
分别生成：
- Results
- Introduction
- Discussion
- Abstract / Title / Keywords

### 阶段04 / 06 / 08 / 10：章节专项质控
对对应章节做结构、证据边界、表达强度与逻辑层面的专项检查。

### 阶段11-14：整稿强化与全局检查
包括：
- 去 AI 味
- 增强研究特异性
- 全文一致性检查
- 文献与引用检查
- 数字统计与事实检查

---

## 轻量后期路线（阶段15-18）怎么理解

这几步不是正文生成的一部分，而是**手稿已接近完成后的轻量后期处理**。

适合：
- 不使用智能体
- 只做基础期刊匹配和投稿前把关
- 希望流程简单、手动可控

如果你希望做更完整的内部审稿模拟、多 reviewer comments、editorial synthesis 与 revision roadmap，请改走：

- [[AI-Paper-Writing-System/08_Agent_System/README|README]]

## 推荐使用顺序

### 如果你还在写作主阶段
按顺序看：
- `阶段00` → `阶段14`

### 如果你已经接近终稿，并且不使用智能体
继续看：
- `阶段15` → `阶段18`

### 如果你已经接近终稿，并且想使用智能体增强后期处理
完成 `阶段00-14` 后，不必强制进入 `阶段15-18`，可以直接查看：
- `[[08_Agent_System/README]]`
