# 阶段03：写 Results

## 目标
生成严格基于图表、图注和方法的 Results 初稿。

## 这一阶段要做什么
1. 准备 Results 专用 prompt
2. 输入图表、图注、Methods、Supplementary、核心数字表
3. 让 AI 按既定结构生成 Results
4. 快速人工核对图文是否对应

## 输入
- Results prompt
- Figures and legends
- Methods
- Supplementary
- Key numbers table
- Results 结构说明

## 输出
- Results 初稿

## 写作要求
- 图表和图注为主要证据来源
- Methods 只用于必要衔接
- 不得编造任何数字或统计结果
- 缺失信息统一使用 `[Data Required]`
- Results 尽量客观，避免 Discussion 式解释
- 避免使用过强因果词

## 为什么 Results 要先写
- Results 是全文证据中心
- Introduction 和 Discussion 都应服务于 Results
- Results 稳定后，全文逻辑更容易统一

## 注意事项
- 每段都应对应明确图表
- 图中没有的信息不要强写
- 先求“稳”和“准”，再求“漂亮”

## 完成标志
- [ ] Results 初稿已生成
- [ ] 图文已做初步人工比对
- [ ] 缺失信息已标记
- [ ] 进入 Results 专项质控
