# Results质控清单

## 1. 证据边界
- [ ] 每一段是否都能对应到明确图表或补充图表
- [ ] 是否写了图上没有、方法里也没有的信息
- [ ] 是否出现过强解释性语言
- [ ] 是否把趋势写成机制
- [ ] 是否把相关性写成因果性

## 2. 内容纯度
- [ ] 是否混入了明显 Discussion 式句子
- [ ] 是否出现“adaptive strategy / mechanism / drives / reveals why”等高风险词
- [ ] 是否有不必要的背景性句子

## 3. 数字与统计
- [ ] 数字是否与图表/关键数字表一致
- [ ] 统计显著性是否有直接依据
- [ ] 无法确认的数字是否保留 `***` 或 `[Data Required]`
- [ ] 是否有明显异常数值或格式问题

## 4. 结构逻辑
- [ ] Results 的分段是否清晰
- [ ] 每段是否围绕一个结果单元推进
- [ ] 段落顺序是否符合图表逻辑
- [ ] 是否存在重复描述

## 5. 输出判断
- [ ] 可以直接进入下一步
- [ ] 需要局部修订
- [ ] 需要重写部分段落
