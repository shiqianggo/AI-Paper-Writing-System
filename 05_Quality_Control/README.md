# Quality Control README

## 目录定位

`05_Quality_Control/` 是本系统的基础质控层。  
该目录用于通过清单式方式，对论文不同章节和关键风险模块进行系统检查。

它的核心作用不是直接代替写作，也不是直接代替多角色审稿，而是作为：

- 写作阶段的基础过滤器
- 定稿前的结构化检查器
- 风险暴露与问题定位工具

---

## 本目录覆盖的质控范围

当前包括：

- Results 质控
- Introduction 质控
- Discussion 质控
- 摘要与标题质控
- 全文一致性质控
- 文献与引用质控
- 数字统计与事实质控
- 投稿前终审清单

---

## 质控层的核心作用

该目录重点解决以下问题：

- 是否混入超出证据边界的表达
- 是否把 Results 写成了 Discussion
- 是否存在结论过强、概括过度的问题
- 是否存在数字、统计值、单位、样本信息错误
- 是否存在文献错配、漏引、伪引
- 是否存在章节之间逻辑脱节
- 是否存在摘要、标题与正文不一致
- 是否存在明显 AI 痕迹与模板化表达

---

## 清单式质控与 Agent 审稿的区别

在系统加入 `08_Agent_System/` 后，需要明确：

### `05_Quality_Control/` 做什么
- 清单式核查
- 逐项排错
- 基础风险过滤
- 局部和全文一致性检查
- 投稿前基础终审

### `08_Agent_System/` 做什么
- 多角色内部审稿模拟
- 审稿意见冲突暴露
- 编辑综合判断
- 修稿路线设计
- 强约束条件下的整稿修订

---

## 二者关系

可以把两者理解为两个层次：

### 第一层：基础质控
由 `05_Quality_Control/` 完成。  
作用是先把明显的结构、表达、文献、数字、事实问题筛掉。

### 第二层：高阶压力测试
由 `08_Agent_System/` 完成。  
作用是模拟更接近真实投稿场景的 reviewer、editor、revision pipeline。

因此，推荐顺序是：

```text
先做 05_Quality_Control
再进入 08_Agent_System
