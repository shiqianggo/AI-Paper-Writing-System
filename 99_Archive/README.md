# Archive

> 本文件夹用于存放不再作为当前主系统默认入口、但仍有保留价值的内容。  
> 它的作用不是“丢弃”，而是**归档、留痕、备查、对比和迭代复用**。

---

## 1. 归档原则

以下内容适合放入 `99_Archive/`：

- 已被新版文件替代的旧版本 Prompt
- 早期 workflow 草稿
- 已废弃但有参考价值的模板
- 某一轮特定项目中使用过、但不再通用的材料
- 历史版本的总原则或质控规则
- 曾经有效但现已过时的实验性写法
- 大模型输出的中间稿、对照稿、失败稿

---

## 2. 本文件夹不放什么

以下内容一般不建议放入归档区：

- 当前仍在使用的正式 Prompt
- 当前版本的标准模板
- 当前版本的质控清单
- 当前项目的核心输入材料
- 仍作为主工作流一部分的活跃文件

这些内容应继续放在对应主目录中，而不是归档区。

---

## 3. 归档的价值

保留归档内容的意义包括：

### 3.1 方便回溯
当你发现新版写法效果不如旧版时，可以快速回查。

### 3.2 方便比较
可以对比：
- 不同 prompt 版本的表现
- 不同模型优化 prompt 的差异
- 不同 workflow 设计的优缺点

### 3.3 方便积累经验
失败版本、冗长版本、越界版本也有价值，  
因为它们能帮助你总结：
- 什么 prompt 容易导致幻觉
- 什么写法容易产生 AI 味
- 什么结构不适合复用

### 3.4 方便系统演化
长期看，`99_Archive/` 是系统迭代的历史记录区。
